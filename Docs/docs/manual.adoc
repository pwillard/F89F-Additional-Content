== Introduction

This document describes the installation, use, and customization of the _OpenRails_ 3D model content included in this package.

[WARNING]
This content is for OpenRails and not intended for use with Microsoft Train Simulator.

The goal of this documentation is to provide clear instructions for:

* Background Information
* Installing the content
* Using the models in OpenRails
* Customizing textures or configurations
* Understanding licensing and distribution rights

=== Background

This is a model of the Bethlehem F89F 89'4" channel side flatcar.  The inspiration for this content was taken from an article in Mainline Modeler Magazine from April 1995.  While the model generally does not fully follow the example provided for the XTTX Trailer Train variant (which is based on a rebuild of a flatcar series formerly used for Auto rack service), it is attempting to match more closely with the follow-up RTTX series of cars that were later marked as "Triple 28/Twin 45" service, which apparently used the earlier flatcar referred to in the article as inspiration for the variable hitch arrangement that could support both 28' and 45' trailers. The XTTX version is the only 4 hitch variant as it was quickly determined that having 4 hitches was not practical.

Many of the RTTX cars began life as tri-level open autoracks cars that were converted to flatcar TOFC service.  The base model here is a the Bethlehem 89'4" channel-side flatcar.  There are other 89' flatcars used for TOFC service that employed the flush deck variant of the car.

[WARNING]
This document also serves as the reference information for the flatcar development kit available separately for those who want access to the texture and model sources.  If you only intend to use this collection in consists there are some sections of this document that can be skipped over.

==== Flexible Loading

The flatcar depicted here has 5 possible hitch locations that can be raised or lowered as needed to load up to 3 28' trailers or 2 45' trailers.  With the RTTX series, the hitches were oriented so the hitches att he extreme ends of the car were closest to the A and B end, essentially flipping the orientation of the trailer on the brake-end of the car.  The original XTTX cars actually had 4 hitches installed and also had tire risers to ensure that no part of the trailers would encounter the hitches as they were rolled on the car.  These cars contained the ramp plates to make roll-on loading possible.  The majority of the RTTX cars would have 3 hitches while the KTTX car series, the "twin 45" would only have the hitches installed at the extreme ends of the car.  Some versions the RTTX cars would have the 45' trailers facing the same direction with a hitch near the center of the car, but this layout seems more rare.


== Installation

To install the model content:

. Extract the ZIP archive
. Copy the provided files into the appropriate OpenRails directories
. Ensure texture and shape files remain in their original structure

The available cars are released in a single folder but you control which variants to install using a consist editor or file.  Customization is as easy as editing the WAG file and referencing different available shapes and related textures.

Each Car and Trailer are individually named inside a WAG file and use appropriately names texture files to go along with them.  

Example directory layout:

----
TRAINSET/
    Specific Decaled Model/
        Flatcar Body  Model S files
		Flatcar Decal FreightAnim Model S files
        Hitch Model FreightAnim Model S files
		Trailer Model FreightAnim Model S files
        DDS texture files for flatcar (body and decal) and FreightAnim Hitches and Trailers
        WAG file, with custom freight anim entries to make it all work

        
----

The WAG files supplied with this content collection are doing all of the work related to what happens with and above the deck of each flat car.  The WAG files themselves are reasonably simplistic other than the FreightAnim() section.  There is certain room for improvement on the physics. Comments welcome.

[TIP]
Maybe someone with better physics expertise can chime in with some recommendations.


== Using the Models

After installation, the model should appear inside OpenRails when the associated consist or rolling stock entry is created and loaded.

Typical usage scenario:

* Customizable Freight rolling stock, of course
* Dynamic loading and unloading of trailers, left up to the user.
* Some of the trailer models contained can be used as Scenery Objects as well.

The models will show up in Shapeviewer and Consist editors, with many different items available, but to be useful, the WAG file must be referenced.

[TIP]
The design of this content is built around the concept of flexibility. The flatcar model itself has no hitches or trailers installed and must be added with FreightAnim statements.  

There are no MT or LD editions of the cars as they remain flexible based on the "programming" in each individual WAG file.  You can certainly use each WAG file as-is, but you can also customize the loading of each  car as desired, or even rename the match MT and LD standardizations.

[NOTE]
These cars utilize NAVS Freight sounds, so you will need to make sure that set is installed as well.

=== Freight Loads 

The freight included with the flexible load options are based mostly on United Parcel Service inter-modal equipment.  The sources for the liveries are available so customization of liveries is also possible.  In fact, if the source shapes are used, you could release additional _FreightAnim()_ trailers as standalone files.  Not included, however, are any of the standard door 28' and 45' as the UPS models all use roll-up doors.

The arrangement of the trailers and hitches is detailed in a section below. 

[NOTE]
A version of the original XTTX car is included in this collection with all four hitches installed, but in a stowed state for anyone wanting to see the original 28' pup trailer/ 45' trailer arrangement.  With the 4 Hitch setup, all trailers faced the same direction. 


[WARNING] 
This collection of content is not necessarily considered to be "plug-and-play".  It has been designed to give the user extreme flexibility.  While it could have been made even more flexible, at some point you have to stop and say "enough" and this gets us to where we are now.

=== Freight Animations

In addition to determining the body style and decal assignments, the `.wag` file configuration determines the hitch installation of raised or stowed configuration in a choice of 3 colors ( Orange, Black or Brown ).  This is done by specific configuration settings in the FreightAnim() section of the WAG file.  Differences in the `.DDS` files are related to the color *and* shadows of stowed versus deployed hitches.

Additionally, the choice of trailer type and trailer owner/markings is completely configurable by calling a specific trailer `.S` file as a loaded FreightAnim shape.

[WARNING]
The current Shape Viewer tools do not respect the _FreightAnim()_ setting of `flip(1)` to flip the orientation of the load by 180 on its Axis.  Also note that ehe axis points for the trailers supplied with this collection is located on the kingpin.  In ShapeViewer, some trailers will seem to have the wrong placement when `flip` is used in the `.WAG` file  It will look fine in OpenRails. 

Here is an example of how this is accomplished (for 3 pup trailers).

[TIP]
The placement of the hitches differs slightly between 28 amd 45 foot trailers, pay attention to the contents of existing WAG files for details.

[source,text]
----

Comment(=============================================)
	ORTSFreightAnims(
		Comment(28' Trailers are offset by 0.2)
		Comment(45' Trailers are offset by 0.3)
		WagonEmptyWeight(36.7t-us)
		
		Comment(=====================================)
		Comment( Hitch #1 : 12.65 -- must use flip   ) 
		Comment(=====================================)

		FreightAnimStatic (
			Flip(1)
			Shape(hitch-deployed-yello.s)
			Offset(0.0 1.045 12.65)
		)

		FreightAnimStatic (
			Flip(1)
			shape(upsz29002.s)
			offset(0.0 1.045 12.85)
			FreightWeight(3.1t)
		)

		Comment(=====================================)
		Comment( Hitch #2 : 5.65)
		Comment(=====================================)

		Comment(=====================================)
		Comment( Hitch #3 : 1.20)
		Comment(=====================================)
		

		Comment(=====================================)
		Comment( Hitch #4 : -3.20)
		Comment(=====================================)

		FreightAnimStatic (
			Shape(hitch-DEPLOYED-black.s)
			Offset(0.0 1.045 -3.2)
		)
		FreightAnimStatic (
			shape(upsz29110.s)
			offset(0.0 1.045 -3.4)
			FreightWeight(3.1t)
		)

		Comment(=====================================)
		Comment( Hitch #5 : -12.65)
		Comment(=====================================)

		Comment(B-END of car)
		FreightAnimStatic (
			Shape(hitch-deployed-yello.s)
			Offset(0.0 1.045 -12.0)
		)
		FreightAnimStatic (
			shape(upsz294099.s)
			offset(0.0 1.045 -12.85)
			FreightWeight(3.1t)
		)

	)
Comment(=============================================)

----

Hopefully this makes sense.  If it doesn't, you could always try to get some assistance from the Trainsim.com Forums or the related Discord chat room.

==== Hitch Locations

The following are the most reasonable hitch locations though they can be adjusted as needed:

[NOTE]
Y axis is UP

[cols="1,1,1,1,1,1,1,1", options="header", frame="all", grid="all"]
|===
|Location 	|Hitch 	|Trailer	|Trailer 	|X |Y 	|Z 			| Note

|B-END      |1      |45'        |28'        |0 |1.045 |12.65 	|
|    		|2 		|    		|28' 		|0 |1.045 |5.65		| XTTX Only
| 			|3 		|45'		|28' 		|0 |1.045 |1.2		| XTTX Only
| 			|4 		|    		| 			|0 |1.045 |-3.2  	|
|A-END 		|5 		|45' 		|28' 		|0 |1.045 |-12.65	| Flip - Not used in XTTX
|===


[TIP]
Trailers should be added to the same offset location as the related hitch, adjusted by approx 0.2m for 28' and 0.3m for 45'. Example: 45' Trailer `Offset(0.0 1.045 -12.95)`

The Layout of the Trailers seems to follow these conventions:

image:layout.png[]

Often times, the the hitches on the extreme ends of the car are fixed in the upright position permanently.


[TIP]
The ORTS Manual for FreightAnim() offset says: "XOffset, YOffset, and ZOffset are the offsets of the shape with respect to its zero position, and are useful to place the shape precisely."  However, it appears that when compared to Blender orientation in this case, offset needs to be specified as `offset(left/right of track center ,Up/Down from track height ,Near/Far along track axis)` for proper positioning. In Blender, Z axis is normally referred to as UP/DOWN so this can lead to some confusion if you are not careful.


Refer to the OpenRails documentation for advanced configuration options related to FreightAnim() and its uses.

== Customization

This content was created with customization in mind.  You can create any combination of trailers and hitches you desire.  Loading a particular livery trailer is as simple as specifying a particular shape file using the placement guidelines in this documentation.  A few examples have been included, some older ones without 3M stripes and older logos and some with stripes and more modern logos.  Customizing new liveries should also be pretty standard provided normal re-skinning methods are followed.

If you want to create a consist that matches a particular ERA, it should be as easy as choosing the flatcar base that best matches and then chosing the approrpiate trailers to go with it.

Again, all defined and controlled by the WAG file.
 
Users may customize the models by:

* Editing textures
* Creating alternative liveries
* Modifying configuration files to select the preferred FreightAnim shape
* Creating new trailer types (the UPS 45' drop bottom trailer has not been modeled, for example)
* Some additional fonts may need to be installed if you make  use of the Affinity files, most of them are from RailSimStuff.com

=== Supplemental Files

For content creators that want some extra content, a github repository has been created to retrieve files that can be modified and shared.  These include:

* Affinity Studio Texture files
* PNG Files
* Blender Source files
* Scripts used while working on content


Link: https://github.com/pwillard/F89F-Additional-Content

* The model uses wheel sets developed by Erick Cantu that were converted to Blender.
* Sounds are supplied by NAVS Freight Sounds -- NAVSFR10.ZIP
* Fonts/Lettering are from Railsimstuff.com
* Reference material comes from hobby magazines,  google searches and rrpicturearchives.net
* Affinity Studio is now free to use and is highly recommended.  The layered masters are available in this format.


[TIP]
Always keep backup copies before making modifications.

The sources are available separately for the Affinity Studio layered master files for the textures as well as the PNG files used.  The lettering on the flat car is not baked into the background image, but rather its located in separate texture with a transparent background eliminating the need to mess with embedded lettering as you normally would when re-skinning.  Changing reporting marks, for example,  would need nothing more than using PAINT.NET or GIMP in a very quick session of minimal rework of the DDS file.

For more detailed or comprehensive updates, you could utilize the Affinity masters, if you are comfortable with that tool.
